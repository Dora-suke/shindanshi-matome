# ノート作成方針

`ノート作成計画一覧.html` の作成・更新ルールを一元化したドキュメント。
「直近N年分を追加して順位を再計算したい」等の要求にも、本書のフローに沿えば再現可能。

最終更新: 2026-04-29

---

## 1. データソース

- **保管場所**: `/Users/nori3/Desktop/1次試験分析/<科目フォルダ>/analysis_data.csv`
- **対象範囲**: H19〜R07（19年分＋R05追試）※ PDF原本も同階層に併存
- **CSVスキーマ**: `年度, 問番号, 設問番号, テーマ要約, 大カテゴリ, 中カテゴリ, 小カテゴリ, キーワード, 図表有無, 計算問題`
- **集計単位**: 原則 **小カテゴリ**（中カテゴリは全体感の確認用）

## 2. スコアリング式（優先順位の決定）

小カテゴリごとに以下で重み付け合算：

| 年度区分 | 重み |
|---|---:|
| 直近7年（現時点では R01〜R07） | 10点 / 問 |
| それ以前（H19〜H30） | 5点 / 問 |

- スコア = `直近7年の問数 × 10 + それ以前の問数 × 5`
- タイブレーク: ①直近5年問数 ②直近7年問数 ③総問数 の順
- 科目別の上位リストは `~/.claude/projects/.../memory/project_10X_note_priority.md` に集約

## 3. 計画一覧HTMLの並び順

- **並び順キー**: **直近5年の出題問数**（降順）／タイトル「直近5年問数順」の根拠
- スコア（§2）は**作成優先順位の判断材料**、HTML表示順は直近5年問数で固定
- 科目内で「既存／更新／新規／追加候補」を**全状態混合**で1〜N位に通し番号

## 4. 参照年数（ノート本文の射程）

項目ごとに **半減期** で可変。HTML の `参照年数` 列に明記。

| 半減期 | 参照年数 | 該当例 |
|---|---|---|
| 長（10年+） | 直近10年 | IE・PERT・乗数効果・基本法理念 等の安定論点 |
| 中（5-7年） | 直近7年 | 生産管理概論・経営革新計画 等 |
| 短（3-5年） | 直近3〜5年 | DX・補助金・サイバーセキュリティ 等の新興論点 |
| 白書系 | 最新版白書 | 開廃業率・規模別データ等の統計 |

判定は項目特性ベースの**判断含み**。迷う場合は中（直近7年）をデフォルトに。

## 5. 状態区分

| 区分 | 定義 |
|---|---|
| 既存 | 既にノート作成済み・参考用に利用 |
| 更新 | 既存ノートに補強版を**別ファイル**で追加（元は残す） |
| 新規 | 新規作成 |
| 追加候補 | 余力次第。スコアは中位だが寿命が長い等の理由で残す |

## 7. ノートの文体・深さ

`~/.claude/projects/.../memory/feedback_note_style.md` に従う：

- 診断士試験向け：技術細部に深入りしない
- 「N大要素」等は**イメージ先行**で示す
- 略語のフルスペル併記は**最初の2回まで**
- 作成は `/note <ID 領域名>` で起動

## 8. 更新フロー（年度追加・順位再計算）

### 8-1. 新年度（例: R08）が公開されたら

1. PDFを `1次試験分析/<科目>/R08_F.pdf` として配置
2. 過去問を分類し `analysis_data.csv` に追記（既存スキーマに合わせる）
3. **直近7年/直近5年の窓をスライド** ＝ R01 が「それ以前」へ降格、R08 が直近に昇格
4. §2 のスコアリングで全小カテゴリを再計算
5. 各科目 memory `project_10X_note_priority.md` の TOP10／TOP15 を更新
6. `ノート作成計画一覧.html` の `直5年` 列・並び順を再生成
7. 大きく順位が変動した項目は「新規」「追加候補」への昇格を検討

### 8-2. 旧年度（例: H17・H18）を遡及追加する場合

1. CSV に追記
2. 重みは **5点側のみ**（直近7年の窓は動かない）
3. 上位は変動小・下位の同点解消に効く程度
4. memory と HTML を再生成

### 8-3. 「直近N年分追加して順位を加味して」依頼への応答

- 「**新年度の追加**」か「**過去年度の遡及追加**」かを最初に確認
- 該当フロー（8-1 / 8-2）を適用
- 重み式（直近7年=10／それ以前=5）は固定。窓の境界が動くだけ
