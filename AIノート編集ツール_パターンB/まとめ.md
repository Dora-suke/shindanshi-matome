## 0. ひとことで何ができる？
- 画面の **左にノート**、**右にAIチャット**。
- 「この表を簡潔に」等と打つと **左のHTMLがその場で書き換わる**。

## 1. 全体像 — 越える壁は2つ
HTMLファイル単体では完結しません。次の2つの仕事が必要でした。

| | 仕事 | たとえ |
|---|---|---|
| **①** | AIを呼ぶ（Claude/Geminiに通信） | シェフに調理させる。電話には**APIキー**が要る |
| **②** | 修正結果をファイルに保存 | 完成した皿を棚にしまう。ブラウザは勝手に上書きできない |

①と②は**別の工程**です（混同しやすい）。流れ：
```
指示を打つ → ①AIに送って修正HTMLを受け取る → ②ファイルに書き込む → 完了
```

## 2. なぜ「パターンB（ローカルサーバー）」にしたか
①②の実現方法には2案ありました。

| | パターンA（サーバー無し） | **パターンB（ローカルサーバー）★採用** |
|---|---|---|
| ①AI呼び | ブラウザが直接たたく | **サーバーが代理でたたく** |
| ②保存 | File System Access API | **サーバーがファイル書込** |
| APIキー | ブラウザに置く（見えうる） | **サーバーに隠せる（安全）** |
| 複数ノート横断 | 苦手 | **得意** |
| RAG（知識庫検索） | ほぼ不可 | **将来追加できる** |
| 起動 | 開くだけ | サーバー起動が要る |

→ 既存の「確認システム（localhost:8765）」と同じ流儀で、**キーを隠せて・ファイルを書けて・将来RAGに伸ばせる** B案を採用。

## 5. モデル選択と「無料枠ゼロ」の注意 ★ハマりどころ
画面のプルダウンでモデルを選べます。**無料で動くのは語尾が「flash」ちょうど**のものだけ。

| モデル | 無料枠 |
|---|---|
| `gemini-2.0-flash` | ◎（既定・確実） |
| `gemini-2.5-flash` | ◯（賢い・出力大きい） |
| `gemini-2.0-flash-**lite**` | ✗ limit:0 |
| `gemini-2.5-**pro**` | ✗ limit:0 |

- `limit: 0` のエラー＝「使い切った」ではなく「**そのモデルに無料枠が無い**」。
- `Please retry in Ns` は1分制限の話。日次 limit:0 は待っても無駄 → **モデルを変えるのが正解**。
- 無料枠は**回数制限あり**＆**入力が学習に使われ得る**ので、機密は載せない。

## 6. お金の話（Claude APIで本格運用したらいくら？）
※自分のAPIで動かす場合の試算。**全文往復**＝HTMLを毎回まるごとAIに送り返す今の方式。

### 1回の修正コスト（Claude Sonnet・全文往復）
| ノートの大きさ | 1回 | 円(≈150円/$) |
|---|---|---|
| 小(10KB) | $0.07 | 約11円 |
| 中(80KB) | $0.54 | 約80円 |
| 大(350KB) | $2.2 | 約330円 |
出力（HTML全文）が費用の約8割。大きいノートほど跳ねる。Opusは約5倍。

### 月100回以上やる場合（自分の使い方）
| 方式 | 月額の目安 |
|---|---|
| 全文往復 × Claude（中ノート） | 約12,000円 |
| 全文往復 × Claude（大ノート多め） | 約30,000円〜 |
| **差分編集＋キャッシュ × Claude** | **約1,000円前後** |
| Gemini無料版（今のサンプル） | 0円（回数制限・品質はClaude未満） |

### 結論（住み分け）
- **重い・徹底的な作り込み** → 今まで通り **Claude Code（定額）**。100回以上やるなら定額が一番安い。
- **軽いちょい直し** → **Gemini無料版の自前ツール**（0円）。
- **自前ツールをClaudeで本格運用** したいなら → **差分編集＋キャッシュを実装**してから（月1,000円前後に収まる）。
- 「全部を自前ツールに移す」必要はない。**定額をメイン＋無料ツールを出先**が、量の多い自分には最適。
