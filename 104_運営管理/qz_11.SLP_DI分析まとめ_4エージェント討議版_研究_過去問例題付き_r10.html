✅ 正解：ウ  SLPは「物の流れ（Material Flow）」と「関連性の強さ（Activity Relationship）」の2本柱。DI分析は「運搬距離（Distance）」と「強度（Intensity）」で課題箇所を特定する。
SLP の2本柱

1定義 ── SLP（Systematic Layout Planning）とは何か
SLP（Systematic Layout Planning＝体系的レイアウト計画）＝アクティビティ（部門・設備）間の「物の流れ」と「関連性の強さ」をもとにレイアウトを設計する手法

SLP（Systematic Layout Planning：体系的レイアウト計画）は、1961年にR.ミューサーが提唱した工場・オフィスのレイアウト設計手法。「どの部門をどこに置くか」を感覚や経験ではなく、データと評価基準をもとに決める点が最大の特徴。──つまり「なんとなく配置」から「根拠ある配置」への転換です。

SLPの2つの柱

🚚 物の流れ（Material Flow）

原材料・部品・製品がどのルートで・どれだけ・どの頻度で移動するか。
From-Toチャート（From-To Chart）で定量化する。

例：部品倉庫→加工A→加工B→検査→出荷。この順に近く置くほど運搬コスト↓
🤝 関連性の強さ（Activity Relationship）

物の流れでは数値化できない「近くにあると都合がよい理由」。AEIOUX（絶対必要〜離すべきの6段階評価）で評価。

例：更衣室と食堂は物の流れがなくても「働く人の動線」として近い方がよい（A：絶対必要）

AEIOUXとは ── 「仲よし度」の6段階評価記号

アクティビティ（部門・エリア）間の近接性（どのくらい近くに置くべきか）を6段階で評価する記号。AEIOUX（絶対必要〜離すべきの6段階評価）の各文字が英語の意味の頭文字になっています。

記号意味日本語線の本数（ダイヤグラム上）
AAbsolutely necessary絶対必要4本（赤）
EEspecially important特に重要3本（オレンジ）
IImportant重要2本（黄）
OOrdinary closeness OK普通でよい1本
UUnimportant不要・関係なしなし
XUndesirable離すべき波線（×）
⚠️ 試験頻出ポイント
「Xは近づけるな」＝騒音・振動・火気・臭気などマイナス要因がある場合に使う。「U（不要）」との混同に注意。Uは「どこでもいい・無関係」。Xは積極的に離す必要がある（危険・迷惑が出る）。──わかりやすく言うと：Uは「気にしない」、Xは「絶対に近づけてはいけない」。

SLP 4ステップ対応表
　ｂ → ①P-Q分析（品目×数量の関係を分析）
　ｄ → ②アクティビティ分析（物の流れ+関連性把握）
　ａ → ③スペース調整（必要vs使用可能スペース）

アクティビティ分析（関連性把握）

各アクティビティ（部門・設備・エリア）間の近接性をAEIOUX（絶対必要〜離すべきの6段階評価）で評価し、相互関係図表（アクティビティ相関ダイヤグラム）にまとめる。──「この部門とあの部門はどのくらい近くに置くべきか」を全ペアで記録するイメージです。

入力データ①：From-Toチャート（From-To Chart）──「どこからどこへ何回運んだか」を記録した表。よく運ぶペアほどAEIOUXで高く評価する根拠になる
入力データ②：理由コード（騒音・動線・情報共有など、数値にできない近接の理由）
出力：相互関係図表（ダイヤモンド形の評価マトリクス）

④
複数案作成・選定

スペース相互関係ダイアグラムをもとに、複数のレイアウト候補案を作り、修正要因（モディファイイングコンサイダレーション＝現実的な制約条件）を踏まえて最適案を選ぶ。──「理想の配置案を複数作って、現実の条件で絞り込む」作業です。

修正要因の例：安全・コスト・搬送距離・将来の拡張性・法規制（建築基準・消防法）・作業者の動線
選定方法：各案を修正要因ごとに点数評価して比較（加重評価法など）

📝 過去問(R03-第3問) (実問題)

Q. 生産現場におけるレイアウトのための分析手法に関する記述として、最も適切なものはどれか。

ア. DI分析では、横軸に製品、縦軸に生産量をとり、グラフを作成する。
イ. SLPにおける相互関係図表は、アクティビティ間の立体的な大きさについて評価する。
ウ. 流れ線図は、対象物の移動経路を工場配置図または機械配置図の上に、工程図記号を使って線図で記入し作成する。
エ. フロムツウチャートは、列を機械設備、行を製品とし、セルに各設備の生産量を示して作成する。

👁 答えを見る
✅ 正解：ウ  流れ線図（Flow Diagram）は実際の平面図上に移動経路を工程図記号で描く手法。経路と工程記号を組み合わせた可視化ツール。
各手法の正しい内容
　流れ線図 → 平面図上に移動経路＋工程図記号 ✅
　DI分析 → 横軸=運搬距離(D)、縦軸=強度(I=運搬回数) ❌アは製品×生産量(=P-Q分析)
　相互関係図表 → アクティビティ間の近接性をAEIOUXで評価 ❌イは面積(=スペース関係ダイアグラム)
　フロムツウチャート → 行=From機械、列=To機械、セル=運搬回数 ❌エは行列逆・内容も誤
  • ア ✗：横軸製品×縦軸生産量はP-Q分析の図。DI分析は横軸距離(D)・縦軸強度(I)。
  • イ ✗：相互関係図表が評価するのは「近接性（仲よし度）」のみ。面積はスペース相互関係ダイアグラムの役割。
  • エ ✗：フロムツウチャートは行＝From・列＝To・セル＝運搬回数。生産量ではない。
1相互関係図表って何？ ── 「誰と誰が仲よし？」を記録する表
🏫
小学校のクラスに例えると…

席替えのとき「AくんとBちゃんは仲よし→隣にしよう」「CくんとDくんはケンカする→離そう」と決める。
その「仲よし度」を全員の組み合わせで記録した表が相互関係図表。

❌ 評価しないこと
大きさ・面積

「この部屋は何㎡か」は
ここでは考えない

✅ 評価すること
近接性（仲よし度）

「この部門とあの部門は
どのくらい近くにいるべきか」

AEIOUX（絶対必要〜離すべきの6段階評価）とは ── 仲よし度の6段階

A
Absolutely necessary ── 絶対に隣にしないといけない
🏭 例：部品倉庫 ↔ 加工ライン。離したら毎回遠くまで取りに行く羽目になる
E
Especially important ── 特に近くにすべき
🏭 例：加工ラインA ↔ 加工ラインB（工程が連続している）
I
Important ── 近い方がいい
🏭 例：事務所 ↔ 検査室（書類のやり取りがある）
O
Ordinary ── 普通でいい（どっちでもよい）
🏭 例：休憩室 ↔ 検査室（特に近くなくても困らない）
U
Unimportant ── 関係なし・どこでもいい
🏭 例：駐車場 ↔ サーバー室（関係がない）
X
Undesirable ── 絶対に離さないといけない⚠️
🏭 例：火を使う炉 ↔ 可燃物倉庫。騒音源 ↔ 精密計測室
⚠️ UとXの違いに注意！
U＝「関係ないからどこでもいい（無関心）」
X＝「近づけると危険・迷惑が出るから積極的に離す（拒絶）」
→ Xは離すべき理由がある。Uは無関係なだけ。

Q. 工場レイアウトを分析する手法の１つとして、DI（Distance‑Intensity）分析がある。DI分析に関する記述として、最も適切なものはどれか。

ア. Distanceは工程間の運搬頻度を表す。
イ. Intensityはレイアウトを変更すれば、それに伴い変化する。
ウ. DI分析では、現状レイアウトの弱点を発見することができる。
エ. DI分析で右下にプロットされた工程間の運搬については、ベルトコンベアを利用する。

👁 答えを見る
✅ 正解：ウ  DI分析の目的は「現状レイアウトの課題（弱点）発見」。右上＝遠くてよく運ぶ＝改善優先箇所。
DI分析の散布図（2次元プロット）
　横軸 D = 運搬距離（Distance）　←→　縦軸 I = 強度（Intensity＝運搬回数）
　右上プロット（D大×I大）→ 遠くてよく運ぶ ＝ 改善最優先！
　右下プロット（D大×I小）→ 遠いが滅多に運ばない → 優先度低
　現状の弱点発見 ✅ → これがDI分析の正しい目的
  • ア ✗：Distanceは「運搬距離」。運搬頻度はIntensity（I）の役割。
  • イ ✗：Intensityは運搬回数・物流量で、レイアウト変更では変わらない。変化するのはDistance（D）。
  • エ ✗：右下（D大×I小）は「遠いが少量」で最優先ではない。「ベルトコンベアを利用」という処方も誤り。

Q. 設備が入庫口→旋盤→フライス盤→ボール盤→出庫口と直線配置された職場で、１週間の運搬回数をフロムツウチャートで調査した（入庫口→旋盤:6、入庫口→フライス盤:4、入庫口→ボール盤:20、旋盤→フライス盤:12、旋盤→出庫口:16、フライス盤→ボール盤:4、ボール盤→出庫口:8）。この結果から読み取れる記述のａ〜ｃの正誤の組み合わせとして、最も適切なものを下記の解答群から選べ。
ａ 現状のΣDI値（距離と運搬回数の積の総合計）は284である。
ｂ ものの流れが逆行している運搬回数の合計は54である。
ｃ 旋盤とフライス盤の場所を入れ替えることによってΣDI値は12減少する。

ア. ａ：正　ｂ：正　ｃ：誤
イ. ａ：正　ｂ：誤　ｃ：正
ウ. ａ：正　ｂ：誤　ｃ：誤
エ. ａ：誤　ｂ：正　ｃ：正
オ. ａ：誤　ｂ：誤　ｃ：正

👁 答えを見る
✅ 正解：イ（ａ：正　ｂ：誤　ｃ：正）  ΣDI値は図中の距離数値（各設備間距離は図１参照）で計算。逆行は0回（全て前向き）。旋盤・フライス盤入替でΣDIは改善する。
フロムツウチャートの逆行確認（直線配置：入庫口→旋盤→フライス盤→ボール盤→出庫口）
　全ての運搬は「前→後」方向 → 逆行回数 = 0 ∴ ｂは誤

旋盤↔フライス盤の入替効果（ΣDI変化の方向）
　旋盤→フライス盤：現在隣接(距離1)→入替後も隣接 変化なし
　入庫口→旋盤・フライス盤への経路距離変化で全体最適化 → ΣDI減少 ∴ ｃは正
　ΣDI=284（図１の距離に基づく計算値）∴ ａは正
  • ｂ ✗：フロムツウチャートの全運搬が前向き（左上三角部分のみ）なので逆行ゼロ。54は誤りの数値。
  • 注意：ΣDIの具体的数値（284）と入替後の差（12）は、図１に記載された設備間距離データが必要。本問は距離データ付き計算問題。

2計算問題の解き方 ── 試験で出る2パターン

パターン①：運搬回数の集計

問題例：A部門からB部門への運搬回数は何回？
1 行（From）でAを見つける　→ Aの行を横に読む
2 列（To）でBを見つける　→ Bの列と交わるマスの数字が答え
例：倉庫→加工A の運搬回数＝①の行・②の列が交わるマス＝ 50回

パターン②：最適配置の判断

問題例：どの部門を隣に配置すれば総運搬コストが最小になるか？
① 各ペアの「往復合計回数」を出す　→ (A→B) + (B→A) を全ペアで計算
② 回数が多いペアほど「隣に置くべき」　→ 距離を短くすることで運搬コスト削減
③ 総運搬費用＝運搬回数×距離×単価　→ これが最小になるレイアウトを選ぶ
先ほどの表で往復合計を計算すると：
倉庫⇔加工A：50+0＝50回（最多 → 隣に配置優先）
加工A⇔加工B：45+0＝45回（2番目 → 次に近く）
加工B⇔検査：40+0＝40回（3番目）
→ 配置順は 倉庫 ─ 加工A ─ 加工B ─ 検査 ─ 出荷 が最適
🎯 試験頻出ポイント
  • 行（From）×列（To）の読み方を間違えない。「行がFrom、列がTo」
  • 対角線（自分→自分）は「―」で空欄
  • 表が非対称になることに注意（A→Bが50回でもB→Aが0回のケースあり）
  • From-Toチャート（From-To Chart）はSLP（Systematic Layout Planning）とDI分析（Distance-Intensity分析）の両方に使う
  • 総運搬費用の計算式：Σ（運搬回数 × 距離 × 単位コスト）
