📝 生産管理は「3ステップ」だけ

工場で何を作るかも、結局はこの3つだけ。

  • ❶ 計画＝何を・いつ・いくつ作る？（生産計画）
  • ❷ 実行＝実際に作る
  • ❸ 統制＝計画通り？モノは？人手は？（生産統制）
👨‍🏫
先生からひとこと「生産管理」って漢字4つ並ぶとうっ…ってなるよね、わかる！ でもね、要は「計画→実行→統制」の3ステップだけ。文化祭の模擬店も同じ。難しくないんだ〜。

🍽️ PERTを「料理」で理解

🍛 夕食3品を最短で作る

料理所要時間前にやること
ご飯を炊く40分なし
カレーを煮る30分野菜を切る（10分）
サラダ5分野菜を切る（10分）
  • ご飯ルート＝40分
  • カレールート＝10＋30＝40分
  • サラダルート＝10＋5＝15分

一番長いルート＝40分。これがクリティカルパスざっくり：プロジェクトで一番時間がかかるルート＝全体の最短完了時間を決める道。

👨‍🏫
先生からひとこと「最長ルートが最短完了？」って混乱するよね。「一番遅い人が決める」と覚えて！ 全員揃わないと食卓は始まらない。最遅組が時間を決める、これだけ！
最長＝最短クリティカルパスの矛盾語 遅いやつが決める全体は遅い人待ち

🖨️ ジョンソン法を「印刷ショップ」で理解

📚 印刷機 → 製本機 の2工程、どの順番で流すと最速？

  • 注文A：印刷20分 / 製本50分
  • 注文B：印刷40分 / 製本10分

ジョンソン法のルール（3手順）

  1. 全部の数の中で一番小さいものを探す → Bの「製本10分」が最小
  2. それは後工程（製本）にある → 後ろから配置 → [_, B]
  3. 残りのAを前に → [A, B]
A→Bの順（最速80分）
時刻0-20分20-60分60-70分70-80分
印刷機A印刷B印刷空き空き
製本機待ちA製本(20-70)B製本

🕑 完了 80分

B→Aの順（110分→30分も損！）
時刻0-40分40-50分50-60分60-110分
印刷機B印刷A印刷空き
製本機ずっと待ちB製本また待ちA製本

🕑 完了 110分

👨‍🏫
先生からひとこと違いの正体は「製本機の暇な時間」！ 短い後工程（B製本10分）を最後にすると、製本機がチョロっと働いて全体が締まる。「後工程が短いやつは後ろ」これが合言葉。
最小が後工程→後ろからジョンソンの鉄則 短い後工程は最後尾暇時間を後ろに集約

🔍 CPM＝PERT＋コスト

PERTとの違い

PERT＝時間管理／CPM（Critical Path Method：クリティカルパス法）＝時間＋コスト管理。CPMではクラッシングざっくり：追加コストをかけて作業期間を短縮することでどれを縮めるか判断。

💰
救命メモ「クラッシング」って横文字怖いけど、「お金で時間を買う」だけのこと。残業代払って早く終わらせる、あれ。

🖨️ ジョンソン法・ディスパッチング 入門

ジョンソン法（2機械フローショップ）

  1. 全作業の中から最小の加工時間を見つける
  2. それが機械1（前）なら→前から割り付け
  3. それが機械2（後）なら→後ろから割り付け
  4. その作業を除いて繰り返す

ディスパッチング（差立て）3大ルール

略語正式名意味
SPTShortest Processing Time加工時間が短い順
EDDEarliest Due Date納期が早い順
FCFSFirst Come First Served先着順
SPT＝平均待ち時間最小短い順で全員待ち減 EDD＝最大遅れ最小納期順で遅刻最小

📝 【1次】試験ポイント：SPT＝平均フロータイム最小／EDD＝最大遅れ最小の特徴セット暗記（H30-9, R02-9, R05-10）。

⚠️ 【1次】引っかけ：「SPTは納期遅れを最小化する」は誤り。納期遅れ最小化はEDD。

📐 PERT/CPMを深掘り

🔁 入門タブの「PERT・CPM」を深掘り。最早・最遅結合点時刻、余裕時間（フロート）の考え方。

最早結合点時刻と最遅結合点時刻

  • 最早結合点時刻：そのイベントに「最も早く到達できる時刻」（前向き計算で前ルートの最大値を採用）
  • 最遅結合点時刻：プロジェクト全体を遅らせない範囲で「最も遅くてよい時刻」（後ろ向き計算で後ルートの最小値を採用）
  • 余裕時間（フロート）＝最遅−最早。クリティカルパス上は余裕ゼロ
前向き＝最早＝MAX合流は最大値で待つ 後ろ向き＝最遅＝MIN分岐は最小値が制約

📝 【1次】試験ポイント：「最早はMAX、最遅はMIN」を計算問題で問われる（H29-6, R04-7）。クリティカルパス上の余裕時間＝0。

⚠️ 【1次】引っかけ：「最早結合点時刻は前ルートの最小値」は誤り。最大値（一番遅い前作業を待つ）。

👨‍🏫
先生からひとこと「合流するイベントは前作業全部が揃わないと始まらない」→だから最大値。「分岐するイベントは後の全作業に間に合わせる」→だから最小値。「合流MAX／分岐MIN」って唱えて！

  2. BOM（部品表）：製品を構成する部品とその個数

🖨️ ジョンソン法 計算ステップ

ジョブ機械1（印刷）機械2（製本）

・余裕時間（フロート）＝最遅−最早。クリティカルパス上は余裕ゼロ

📊 山積み・山崩し（余力管理の調整）

📝 余力管理の実践

  • 山積み：作業負荷を時系列に積み上げて見える化（棒グラフ）
  • 山崩し：能力を超えた山を、余裕日や別日に振り分けて平準化
⛰️
救命メモ「山」の比喩そのまま。負荷の山が能力の高さを越えたら、余裕日に崩して引き伸ばす。引っ越しの段ボールを部屋ごとに分ける感覚！

📝 【1次】試験ポイント：山積み＝可視化／山崩し＝平準化の役割分担（R03-4）。
