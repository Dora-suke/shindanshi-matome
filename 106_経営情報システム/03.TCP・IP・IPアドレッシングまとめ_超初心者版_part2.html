📍 ステップ1：IPアドレスは「2進数32桁」で書ける

🎯 数字を1個ずつ「8桁の2進数」に変換しただけ

IPアドレスは 192.168.1.10 のように4つの数字。実はこれ、0と1の並び（2進数）に変換できます。

192.168.1.10
↓ 各数字を「8桁の2進数」に直すと…
11000000.10101000.00000001.00001010

1個の数字＝8桁の0と1。それが4個並ぶので、合計32桁＝32ビット。「IPv4は32ビット」って言われるのはこの意味です。

👨‍🏫
先生からひとこと「2進数なんて分からない！」って大丈夫、計算しなくていいです。覚えるのは「IPアドレスは内部的に0と1が32桁並んだもの」という事実だけ。これが分かれば次のCIDRがスッキリ理解できるよ。

🧐 ステップ2：CIDR「/N」は「先頭何桁が固定か」

📝 CIDR（Classless Inter-Domain Routing：クラスレス・ドメイン間経路制御） とは

CIDR（シーアイディーアール）は 192.168.1.0/24 みたいに書く記法。スラッシュの後ろの数字が「先頭から何桁の0/1が固定されているか」を表します。

  • /24 ＝ 先頭24桁が固定／残り8桁が自由
  • /16 ＝ 先頭16桁が固定／残り16桁が自由
  • /8 ＝ 先頭 8桁が固定／残り24桁が自由

「自由な桁」が端末の番号になります。だから自由な桁が多いほど、収容できる端末が多い。

💡 覚えるのはこれだけ：
「/Nの数字が大きい＝固定が多い＝範囲が狭い＝端末少ない」
「/Nの数字が小さい＝固定が少ない＝範囲が広い＝端末多い」
大きい/N＝狭い/28＝14台小さい/N＝広い/8＝1,677万台
👨‍🏫
先生からひとことここ、数字と範囲の関係が逆だから引っかかりやすいの！「大きい数字＝小さい範囲」、これ口に出して3回唱えとこ。「/24より/28のほうが狭い」は「ニジュウヨンよりニジュウハチのほうが家が小さい」って覚えてもOK。

🏢 ステップ3：プライベートIPの クラスA/B/C と CIDR の対応

Part1で出てきた クラスA／B／C がそれぞれ /8、/12、/16 になる理由を見ていきます。「固定の桁数」を数えるだけ。

🔵 クラスA → /8

範囲：10.0.0.0 〜 10.255.255.255

  • 1組目「10」が固定（8桁分）
  • 2〜4組目は0〜255まで自由（24桁分）

つまり固定が8桁 → /8。自由な24桁で 2の24乗 ≒ 約1,677万台を収容できる（大企業向け）。

🟠 クラスB → /12（ここがちょっと難しい）

範囲：172.16.0.0 〜 172.31.255.255

  • 1組目「172」が固定（8桁分）
  • 2組目は16〜31の範囲だけ＝上の4桁だけ固定、下の4桁は自由
  • 3〜4組目は自由

合わせて固定が 8＋4 ＝ 12桁 → /12。自由な20桁で約100万台を収容（中規模向け）。

👨‍🏫
先生からひとこと「16〜31の範囲ってなんで4桁固定？」と思った？ 16〜31の数字を2進数で書くとぜんぶ「0001◯◯◯◯」で始まるから。前の4桁が共通＝固定なんだ。試験では細部不要、「クラスBは/12」だけ覚えればOK！

🟢 クラスC → /16

範囲：192.168.0.0 〜 192.168.255.255

  • 1組目「192」と2組目「168」が両方固定（8＋8 ＝ 16桁）
  • 3〜4組目は自由（16桁）

固定が16桁 → /16。自由な16桁で約6万5千台（家庭・SOHO向け）。

A=/8（10）大企業B=/12（172.16〜31）中規模C=/16（192.168）家庭

🧮 ステップ4：「使える台数」の計算公式

📋 公式：使える台数 ＝ 2 のホスト部ビット数 乗 − 2

言い換えると：2^(32 − /Nの数字) − 2

なんで「−2」するの？ → 2つの番号は予約されているからです（次の例えで説明）。

💡
なぜ −2？（マンションで例え）
自由な8桁なら、番号は 2^8 ＝ 256通り作れます。でも住人に渡せる番号は254個。なぜ？

・番号00000000（全部0） → 「このマンション全体」を指す番号として予約（住人には使わせない）
・番号11111111（全部1） → 「マンション内の全員にお知らせ」用に予約

この2つを引くから「−2」。256 − 2 ＝ 254台。

📊 よく出る計算結果（試験で出るのはこの3つだけ）

CIDR計算式使える台数規模感
★ /242^(32−24) − 2 ＝ 2^8 − 2254 台小オフィス
★ /262^(32−26) − 2 ＝ 2^6 − 262 台1/4フロア
★ /282^(32−28) − 2 ＝ 2^4 − 214 台少人数
/252^7 − 2126 台半フロア
/272^5 − 230 台小部署

📝 【1次】試験ポイント：R02問9・R04問11で出題。★の3つ（/24=254／/26=62／/28=14）のどれかが出ます。

/24＝ニーゴーヨン254台/26＝ロクニ62台/28＝イチヨン14台
👨‍🏫
先生からひとこと「2の何乗とか苦手…」って大丈夫！ ★の3つの「答え」だけ丸暗記でOK。試験会場で公式を思い出せなくても、「/24は254、/26は62、/28は14」って言えれば正解にたどり着けます。
⭐ 今日のこれだけ覚えて（詳細①CIDR）
  • ❶ CIDR「/N」＝先頭から何桁が固定か。大きい数字＝狭い範囲（逆なので注意！）
  • ❷ クラスA=/8（10）／クラスB=/12（172.16〜31）／クラスC=/16（192.168）
  • ❸ 使える台数：/24=254／/26=62／/28=14（−2はマンションの「全体」と「全員宛」が予約だから）
📝 お疲れさまでした！ 続けて「🔍 詳細②IPv6/ポート」タブへ。

🌍 IPv4（Internet Protocol version 4：IP第4版） vs IPv6（Internet Protocol version 6：IP第6版）

📍 ひとことで言うと「桁数の違い」

IPv4は32桁、IPv6は128桁の0と1で書きます。桁数が増えると使える住所の数がケタ違いに増えます。

項目IPv4IPv6
アドレス長32ビット128ビット
表記10進数4組（例：192.168.1.1）16進数8組（例：2001:db8::1）
個数約43億（枯渇）事実上無限
NAT（Network Address Translation：アドレス変換）ほぼ必須不要

📝 【1次】試験ポイント：H27問8・R01問12で出題。IPv6は128ビットの暗記が必須。

⚠️ 【1次】引っかけ：「IPv6ではNATが必須」は誤り。正しくはNATは不要（IPがあり余るため）。

IPv4＝32ビット10進4組IPv6＝128ビット16進8組／NAT不要
👨‍🏫
先生からひとことIPv4が32ビット、IPv6が128ビット。「6だから後ろに6つの数字」……は嘘だけど、「v4の4倍が128」って覚えると忘れない！ ポイントは「IPv6はアドレスが余りまくるからNATいらない」。試験で「IPv6でもNAT必須」って書いてあったら×、これだけ！

🚪 ポート番号（完全版）

💡 ★6つだけ覚えればOK。
ポート番号＝建物の「◯◯窓口」の番号。確実型＝TCP（Transmission Control Protocol：通信制御プロトコル／書留）／速い型＝UDP（User Datagram Protocol：ユーザデータグラムプロトコル／普通郵便）。
表内略語の正式名：FTP=File Transfer Protocol／SMTP=Simple Mail Transfer Protocol／DNS=Domain Name System／DHCP=Dynamic Host Configuration Protocol／HTTP=HyperText Transfer Protocol／POP=Post Office Protocol／IMAP=Internet Message Access Protocol／HTTPS=HTTP Secure
サービスポート型ひとことで何？
FTP20/21確実型ファイル転送
★ SMTP25確実型メールを送る
★ DNS53速い型名前⇔IP変換（電話帳）
DHCP67/68速い型IPアドレスを自動配布
★ HTTP80確実型Webページ
★ POP110確実型メール受信（DL型）
★ IMAP143確実型メール受信（残置型）
★ HTTPS443確実型Webページ（暗号化）

📝 【1次】試験ポイント：頻出は★の6つ。HTTP=80／HTTPS=443／SMTP=25／POP=110／IMAP=143 はセット暗記。

ハチマル＝WebHTTP=80ヨンヨンサン＝セキュアHTTPS=443ニーゴ＝送信SMTP=25イイレイ＝POPPOP=110イーヨミ＝IMAPIMAP=143（イ・ヨ・ミ）
👨‍️🏫
先生からひとこと11個全部は厳しいよね。★の6つだけに絞ろう！特に試験で出るのはHTTP=80・HTTPS=443・SMTP=25の3点セット。残りの POP=110、IMAP=143、DNS=53 は「なんとなく見たことある」レベルで合格できるよ。Telnet とか FTP-Data とかは無視してOK。

💡 応用論点

【1次の横断】ファイアウォール設定

Webサーバーでは80／443番の窓口だけ開けて、他は閉じるのが基本。「開いている窓口＝攻撃される入口」なので、使う窓口以外は全部閉めるのが鉄則。

【1次の横断】クラウド（AWS等）でも同じ用語

クラウド上に「自分専用のネットワーク区画」を借りる際にも、本書で学んだ CIDR・サブネット・NAT がそのまま使われます。オンプレで覚えた知識がクラウドでも通用するということ。

⭐ 今日のこれだけ覚えて（詳細②IPv6/ポート）
  • ❶ IPv6=128ビット・NAT不要。「IPv6でNAT必須」は引っかけ（誤り）
  • ❷ 頻出ポート★6つ：HTTP=80／HTTPS=443／SMTP=25／POP=110／IMAP=143／DNS=53
  • ❸ Webサーバーは 80/443 だけ開ける／クラウドでも CIDR・サブネット・NAT は同じ
事例1

花屋A店：新店舗で Wi-Fi を導入したい

与件：レジ2台、事務用PC3台、従業員スマホ4台、お客様用フリーWi-Fi。合計10台＋α。

解決：家庭用Wi-Fiルータ1台で足りる。内部はプライベートIP（192.168.1.〜）、外向けはNAPT（Network Address Port Translation：ポート付きアドレス変換）で1つのグローバルIPを共有。お客様向けは別SSIDに分離してセキュリティ確保。

学び：規模が小さければ市販ルータ1台で構築可能。NAPTでグローバルIPコスト削減。

👨‍🏫
先生からひとこと診断士試験はこういう「中小企業の現場でどう使う？」が肝。「規模が小さい＝市販ルータ＋NAPTで十分」って結論は試験でも実務でも超頻出のパターン。難しく考えず、「家のWi-Fi構成と一緒」って思えばOK！
事例2

製造業B社：本社と支店をつなぎたい

与件：本社（本州）と支店（北海道）で在庫・顧客情報を共有したい。

解決：VPN（Virtual Private Network：仮想プライベート網）インターネット経由で安全な専用線を作る技術で両拠点を暗号化トンネルで接続。お互いのプライベートIPで通信可能に。専用線より安く、インターネット経由より安全。

事例3

サービス業C社：Webサイトが頻繁に落ちる

与件：自前サーバーでWeb公開。アクセス集中で応答しなくなる。

解決：クラウド（AWS等）に移行。サーバーがダウンしても自動で別サーバーに切り替わる設計（冗長化）。CIDR・サブネットを設計し、既存ネットワーク知識がそのまま使える。

小規模＝市販ルータ＋NAPT拠点間＝VPNアクセス集中＝クラウド
⭐ 今日のこれだけ覚えて（事例）
  • ❶ 小規模オフィス → 市販Wi-Fiルータ1台＋NAPTでOK
  • ❷ 離れた拠点を繋ぐ → VPN（暗号化トンネル）
  • ❸ アクセス集中・サーバー落ち → クラウド移行＋冗長化

📱 家のWi-Fiルータの中で起きていること

スマホで YouTube を見始めると…

  1. スマホが「youtube.com のIPアドレスは？」とルータに問合せ（DNS／UDP・ポート53）
  2. ルータがインターネットのDNSサーバーに問合せてIPを取得（ここまでUDP）
  3. スマホが取得したIPのYouTubeのWebサーバにTCP接続（ポート443でHTTPS）
    ※ ②までのDNS（UDP）と、③のWeb通信（TCP）は別のプロトコル。プロトコルがここで切り替わる！
  4. ルータが NAPT でスマホのプライベートIPを自分のグローバルIPに変換
  5. 動画データがパケットで小分けに送られて、ブラウザで再生

数秒の裏で、DNS・TCP・HTTPS・NAPT が全部動いている！

🌍 IPv4が足りなくなった話（2011年枯渇）

世界のIPv4在庫は2011年に枯渇。43億個では世界人口70億と機器多数に足りない。対策：

  • NAPTで1つのIPを多数端末で共有（延命策）
  • IPv6に移行（128ビット、事実上無限）
👨‍️🏫
先生からひとこと「2011年に枯渇」って覚えなくていいよ！ 大事なのは「足りなくなったから対策してる」ってこと。対策2つ：NAPTで延命／IPv6で根本解決。試験では「IPv4枯渇への対策は？」って聞かれたらこの2つを答えればOK！
⭐ 今日のこれだけ覚えて（実例）
  • ❶ YouTube視聴の裏で動くのは DNS（UDP）→TCP接続→HTTPS→NAPT。DNSはUDP、その後のWeb通信はTCPと切り替わる
  • ❷ IPv4枯渇への対策＝NAPT＋IPv6（この2つセット）

🧭 図解1：OSI（Open Systems Interconnection：開放型システム間相互接続） 7層と代表プロトコル

層名前代表
7アプリHTTP/HTTPS/SMTP/POP/IMAP/DNS
6プレゼンSSL/TLS（Secure Sockets Layer／Transport Layer Security）
5セッション—
4トランスポートTCP/UDP
3ネットワークIP
2データリンクイーサネット
1物理ケーブル/電波

通信プロトコルがどの層で動くかの早見表。試験で頻出の3点はHTTP=7層／TCP/UDP=4層／IP=3層。「Webページを取りに行く（HTTP）」「確実か速さか決める（TCP/UDP）」「住所と経路を決める（IP）」が、それぞれ別の層の仕事として整理されています。

🧭 図解3：CIDRと使えるホスト数

CIDRホスト数規模感
/24254小オフィス
/26621/4フロア
/2814少人数

CIDR「/X」の数字が大きいほど範囲は狭く（使える端末が少なく）なります。公式は 2^(32−X) − 2（−2はネットワークアドレスとブロードキャストの予約分）。/24=254、/26=62、/28=14 はサブネット計算問題で頻出。

👨‍️🏫
先生からひとこと図解は試験前日の最終チェックに使うのがオススメ！ ここに載ってる3つの図さえ脳に焼き付ければ、TCP/IP分野の半分は取れる。OSI 3点（HTTP=7／TCP=4／IP=3）、プライベートIP 3つ（10／172.16〜31／192.168）、CIDR 3つ（254／62／14）。3×3＝9個覚えるだけ！
