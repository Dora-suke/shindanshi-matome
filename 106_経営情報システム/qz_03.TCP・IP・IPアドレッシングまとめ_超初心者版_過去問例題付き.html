🌐 インターネットってそもそも何？

💡
教科書 4-4 より
インターネット（Internet）を直訳すると「ネットワークの間のネットワーク」。世界中のLAN（家や会社の小さな網）が相互に接続された巨大なネットワークです。

📖 3つのキーワード（教科書序盤）

  • TCP/IP＝インターネットの共通ルール（プロトコル）。世界中がこのルールを守ることで繋がる
  • ICANN（Internet Corporation for Assigned Names and Numbers）＝IPアドレスとドメイン名を管理する非営利団体
  • WWW（World Wide Web）＝インターネット上でWebページを参照できる仕組み。略して「Web」

📝 【1次】試験ポイント：H22-8、H21-6、H18-10で出題。「インターネット＝LANの集合・TCP/IPベース」「ICANN＝管理団体」「WWW＝Web閲覧の仕組み」の3点。

インターネット＝LAN同士のネットICANN＝管理団体WWW＝Web
👨‍🏫
先生からひとこと「インターネット」って当たり前すぎて改めて聞かれると困るよね。「家や会社のLAN（小さな網）が世界中繋がったもの」これだけで OK。ICANNは試験でちょこちょこ顔出すから「IP・ドメインの管理団体」って覚えとこ！
💡 かんたん解説

インターネットは「世界中のLAN（小さなネット）がルールを守って繋がった巨大な網」。TCP/IPはそのルール、ICANNはルールを管理する団体、WWWはその上でWebページを見る仕組み。

📚 TCP/IP プロトコルって何？

💡
教科書 4-4 より
TCP/IP プロトコルは正確にはIP と TCP の2つのプロトコルから構成。両者は別々の階層で動作します。
プロトコルは階層構造になっており、情報を送る側は上位層から下位層へ、受け取る側は下位層から上位層へ流れます。手紙で言えば「宛名を書く（上位）→ 航空便で運ぶ（下位）」のような分業。

🏢 OSI 基本参照モデル（7階層）

OSI（Open Systems Interconnection）基本参照モデル

ISO（国際標準化機構）が定めた7階層のプロトコル標準モデル。実在のプロトコルではなく、設計のお手本。

層名称定義する内容
第7層アプリケーション層提供する通信サービスの種類（HTTP・FTP・メールなど）
第6層プレゼンテーション層データの形式（文字コード変換など）
第5層セッション層通信の開始から終了までの管理
第4層トランスポート層通信における信頼性確保（TCP/UDP）
第3層ネットワーク層異なるネットワーク同士の中継方法（IP）
第2層データリンク層限られた範囲内での機器間接続
第1層物理層物理的な接続方法（ケーブルなど）
7層＝アプセトネデブ上から順
👨‍🏫
先生からひとこと7層って多くて覚えるのキツいよね…でも語呂「アプセトネデブ」を口に出して3回唱えればもう大丈夫！ 上から「アプリ」始まり、下が「物理」って向きだけ間違えないように！
💡 かんたん解説

OSI 7層は「通信の仕事を7つの分担に分けた設計図」。上が人間寄り（アプリ）・下がケーブル寄り（物理）。語呂「アプセトネデブ」で順番を丸ごと覚えよう。

📦 OSI 7層（上が第7層・下が第1層）
7 アプリケーション層 ← HTTP・FTP・SMTP
6 プレゼンテーション層 ← 文字コード変換
5 セッション層 ← 通信開始〜終了の管理
4 トランスポート層 ← TCP・UDP　★頻出
3 ネットワーク層 ← IP　★頻出
2 データリンク層 ← MACアドレス
1 物理層 ← ケーブル・電気信号
📝 過去問エッセンス(H21-15系) (エッセンス)

Q. OSI基本参照モデルに関する記述として、最も適切なものはどれか。

  • ア. TCP/IPのトランスポート層に対応するのはOSI第4層（トランスポート層）である
  • イ. HTTPはOSI第3層（ネットワーク層）に相当するプロトコルである
  • ウ. ルータはOSI第2層（データリンク層）で動作する
  • エ. OSI第1層は「セッション層」と呼ばれる
👁 答えを見る
✅ 正解:ア  TCPはOSI第4層（トランスポート層）に相当。HTTP=7層、TCP=4層、IP=3層が最重要3点。
  • イ ✗:HTTPはアプリケーション層（第7層）で、第3層はIPが担当する
  • ウ ✗:ルータはIPアドレスを扱う第3層（ネットワーク層）で動作する
  • エ ✗:第1層は「物理層」。セッション層は第5層
✏️ 練習問題｜OSI各層の役割

Q. OSI基本参照モデルの各層に関する記述として最も適切なものはどれか。

  • ア. 第5層（セッション層）はデータの形式変換・暗号化を担当する
  • イ. 第7層（アプリケーション層）はHTTP・SMTP・FTPなど利用者に近い通信を担当する
  • ウ. 第3層（ネットワーク層）はポート番号でアプリケーションを識別する
  • エ. 第1層（物理層）はMACアドレスを使って機器間通信を行う
👁 答えを見る
✅ 正解:イ  第7層（アプリケーション層）はHTTP・SMTP・FTPなど利用者に直接関わるプロトコルを扱う。
  • ア ✗:データ形式変換・暗号化は第6層（プレゼンテーション層）の役割
  • ウ ✗:ポート番号でアプリを識別するのは第4層（トランスポート層・TCP）
  • エ ✗:MACアドレスを使うのは第2層（データリンク層）。第1層は電気信号の物理接続

  • イ. OSI第3層で動作し、異なるネットワーク間のルーティングを行う

🔀 OSI と TCP/IP の対応

TCP/IPプロトコルは、OSI 7層モデルに対応させると4階層になります。

OSI 7層TCP/IP 4層代表プロトコル
第5〜7層（セッション・プレゼン・アプリ）アプリケーション層HTTP／FTP／SMTP／POP／IMAP
第4層（トランスポート）TCPTCP／UDP（ポート番号でアプリ識別）
第3層（ネットワーク）IPIP（IPアドレスでコンピュータ識別）
第1〜2層（物理・データリンク）ネットワークインタフェース層MACアドレスでNIC間通信

  • イ. TCP/IPは7層構造であり、OSIと完全に一致する

📍 IP（Internet Protocol）＝インターネットの基本

💡
教科書 4-4 最頻出セクション
IP は通信する際のデータの最小単位＝パケットを、相手に送信する役割。パケットにはデータ本体と宛先などのヘッダがある。手紙で言えば封筒の宛名がヘッダ、中身がデータ。

🏠 IPアドレス＝ネット上の住所

パケットの宛先は IPアドレスで記載。ネットワーク機器に一意の番号として割り当てる。

現在主流の IPv4 は4つに区切られた32ビットの数値。例：192.168.36.123（10進表記）

IPアドレス＝建物の住所IPv4＝32ビット・10進4組

🏪 サブネットマスク（ネット部とホスト部の境界）

サブネットマスクは IPアドレスをネットワーク部とホスト部に分けるための数値。
2進法で 1の部分がネットワーク部、0の部分がホスト部。

例：192.168.10.0／255.255.255.0（24ビット） → 先頭24ビットがネットワーク部、残り8ビットがホスト部。

CIDR表記では 192.168.10.0/24 と書く（/24 = 「先頭24ビットがネットワーク部」）。

👨‍🏫
先生からひとことサブネットマスクって難しそうだけど、「住所のうちどこまでが街の名前で、どこからが家の番号か」を分ける目印だと思って！ /24 なら「先頭3組（24ビット）が街の名前」って意味。

  • イ. IPv6のアドレスは10進数4組で表現する

📖 ドメイン名と DNS

IPアドレスは数字の羅列で覚えにくいので、人間に分かりやすいドメイン名を付ける。
例：「studying.jp」「google.com」「amazon.co.jp」

ブラウザに「google.com」と打つと、裏で DNS が「IPアドレスは142.250.xxx.xxx」と教えてくれる。

📞 グローバルIP と プライベートIP

グローバルIP

全世界からのアクセスで使うため、重複しない一意の番号。

プライベートIP

社内ネットワークなど外部接続が要らない場合に内部だけで使う。内部で一意であればOK。

グローバル＝世界共通の住所プライベート＝社内の内線番号

🔄 NAT と NAPT（IPマスカレード）

NAT（Network Address Translation）

プライベートIPをグローバルIPに1対1で変換する仕組み。社内のPCが外部ネットへアクセスする時に使う（ルータが提供）。
5台同時にアクセスするには、グローバルIPが5個必要。

NAPT（Network Address Port Translation）＝IPマスカレード

プロバイダから貰えるグローバルIPは1個のことが多い。NAPTは1個のグローバルIPを複数端末で共有する仕組み。
ルータが各端末に異なるポート番号を割り当てて識別する。

NAT＝1対1NAPT＝多対1IPマスカレードとも呼ぶ

📝 【1次】試験ポイント：R1-11、H29-11 出題。「1つのグローバルIPで複数端末を同時接続＝NAPT」。

💡 かんたん解説

NATは「社内のPC 1台につきグローバルIP 1つ」の1対1変換。NAPTは「グローバルIP 1つをポート番号で振り分けて複数台が同時使用できる」多対1変換。家庭のWi-Fiルーターはほぼ全部NAPT。

Q. NAPTに関する記述として最も適切なものはどれか。

  • ア. 1つのグローバルIPアドレスをポート番号で振り分けることで複数台の端末が同時にインターネットへ接続できる
  • イ. プライベートIPとグローバルIPを1対1で変換するのがNAPTである
  • ウ. NAPTでは複数のグローバルIPアドレスが必要である
  • エ. NAPTはMACアドレスを使って各端末を識別する
👁 答えを見る
✅ 正解:ア  NAPTは1つのグローバルIPをポート番号で振り分け複数端末が共有。家庭のWi-Fiルーターはほぼこの方式。
  • イ ✗:1対1でプライベートIPをグローバルIPに変換するのはNAT（NAPTではない）
  • ウ ✗:NAPTの目的は1つのグローバルIPで複数端末が使えること
  • エ ✗:NAPTはポート番号で端末を識別する。MACアドレスはデータリンク層で使用
✏️ 練習問題｜NATとNAPTの違い

Q. NATとNAPTの違いに関する記述として正しいものはどれか。

  • ア. NATもNAPTも1つのグローバルIPで複数端末が同時接続できる
  • イ. NATはプライベートIPを1対1でグローバルIPに変換し、NAPTはポート番号も使って多対1変換を行う
  • ウ. NAPTは1対1変換でNATが多対1変換である
  • エ. NATはDNSサーバが行う機能でありNAPTはルータが行う機能である
👁 答えを見る
✅ 正解:イ  NATは1対1変換、NAPTはポート番号で多対1変換。どちらもルータが担うがNAPTの方が実用的。
  • ア ✗:NATは1対1変換のため複数端末分のグローバルIPが必要。多対1はNAPTのみ
  • ウ ✗:NATとNAPTの説明が逆。NATが1対1、NAPTが多対1（ポート識別）
  • エ ✗:NATもNAPTもルータが行う機能。DNSはドメイン名からIPアドレスへの変換

🔁 DHCP（IP自動割当）

DHCP（Dynamic Host Configuration Protocol）＝ネットワークに接続するコンピュータにIPアドレスを自動的に割り当てるプロトコル。
DHCPがなければ、PC一台ずつ手動でIP設定が必要。これがあるおかげで、Wi-Fiに繋いだら自動でネットが使える！

DHCP＝IPの自動配布引越し屋さん

🚪 TCP（Transmission Control Protocol）

💡
教科書 4-4 より
IP では IPアドレスを宛先としてパケットを特定のコンピュータに送信できました。
TCP はさらにポート番号を使って、コンピュータ上のアプリケーション（プログラム）を特定します。
また TCP は到着の確認やエラーの訂正を行い、通信を正確にします。

🏢 ポート番号＝建物の部屋番号

IPアドレスが建物の住所とすれば、ポート番号は部屋番号のようなイメージ。

ポート番号は0〜65535の数値。よく使われるプロトコルにはあらかじめ決まったポート番号がある。
例：HTTPプロトコルのポート番号は80（教科書記載）

IP＝住所ポート番号＝部屋番号HTTP=80
👨‍🏫
先生からひとこと「住所だけだと建物のどこ？」って分からないよね。「住所＝IP、部屋番号＝ポート」って覚えるとイメージしやすい！ HTTP=80 だけは教科書記載の最重要ポート番号。これだけ押さえとけば十分！
💡 かんたん解説

IPアドレスで「どのコンピュータか」を特定し、ポート番号で「そのコンピュータの中のどのアプリか」を特定する。マンションで言えばIPが住所・ポートが部屋番号。

📱 UDP（教科書補足）

UDP（User Datagram Protocol）

OSI モデルでは TCP と同じトランスポート層に相当。
コネクションレス型のデータ転送で、到着確認しない。
→ TCPより信頼性は低いが、転送速度が高い。

TCP＝信頼性高（書留）UDP＝速い（投げっぱなし）
💡 かんたん解説

TCPは「書留郵便」のように届いたか確認するので信頼性が高い。UDPは「チラシのポスティング」のように確認せず投げっぱなしなので速い。動画や音声リアルタイム通信はUDPが向いている。

📨 アプリケーション層のプロトコル

💡
教科書 4-4 より
TCP/IP は通信の基本機能を提供しますが、その上のアプリケーション層には目的に応じた様々なプロトコルがあります。

📖 HTTP（Hyper Text Transfer Protocol）

Webページの元データ＝ハイパーテキストを送受信するプロトコル。
Webブラウザでアクセスする際、「http:」または「https:」から始まる URL（ネットワーク上のリソースの場所を表す識別子）を指定。

📂 FTP（File Transfer Protocol）

ファイルを転送する際のプロトコル。大きなファイルのDLや、Webページを Webサーバへ転送する時に使う。

Q. 電子メールのプロトコルに関する記述として最も適切なものはどれか。

  • ア. SMTPはメールの受信に使うプロトコルで、送信にはPOPを使う
  • イ. POPはサーバにメールを保存したまま特定のメールのみ取り出せる方式である
  • ウ. SMTPはメール送信に使用され、POPはすべてのメールをダウンロードして受信する方式である
  • エ. IMAPはサーバからメールを全件ダウンロードし、サーバ側のメールを削除する
👁 答えを見る
✅ 正解:ウ  SMTPはメール送信、POPは全件ダウンロード受信（サーバから削除）、IMAPはサーバ保存のまま選択受信。
  • ア ✗:SMTPが送信、POPが受信。SMTPとPOPの役割が逆
  • イ ✗:タイトルを見て選択受信できるのはIMAP。POPは全件DL方式
  • エ ✗:全件DLしてサーバから削除するのはPOP。IMAPはサーバに残したまま
✏️ 練習問題｜IMAPとPOPの使い分け

Q. IMAPとPOPの違いに関する記述として最も適切なものはどれか。

  • ア. POPはサーバにメールを保存したまま複数端末で同期できる
  • イ. IMAPはすべてのメールをクライアントにダウンロードしサーバから削除する
  • ウ. IMAPもPOPもメール送信に使われるプロトコルである
  • エ. IMAPはサーバ上にメールを保存したままタイトルを確認し必要なメールだけDLできるため複数端末での利用に向く
👁 答えを見る
✅ 正解:エ  IMAPはサーバにメールを残したまま選択受信できるため、スマホとPCで同じメールを共有できる。
  • ア ✗:複数端末で同期できるのはIMAP。POPはDL後サーバ削除のため他端末で見られない
  • イ ✗:全件DLしてサーバから削除するのはPOPの動作
  • ウ ✗:IMAPもPOPも受信プロトコル。送信はSMTPが担当する

🌐 WWW と HTML

WWW（World Wide Web）

インターネット上でWebページを参照できる仕組み。Webブラウザに URL を入力すると Webページを読み込んで画面に表示。

Webページを公開するには Webサーバが必要。Webサーバは Webブラウザからの要求に応えてHTMLファイルなどを送信するソフトウェア。

HTML（HyperText Markup Language）

Webページを作るための言語。文字だけでなくリンク・見出し・画像を含めた Webページを記述。
HTML で作成したページは FTP などで Webサーバに転送して公開。

POP と IMAP の違い（教科書最重要）

  • POP：全てのメールがクライアントにダウンロードされる。サーバからは削除
  • IMAP：タイトルだけ見て、ダウンロードするメールを選択できる方式。サーバに保存されたまま。モバイル環境などに便利
👨‍️🏫
先生からひとことPOP と IMAP、どっちが何？って混乱しやすい！ 覚え方：「IMAPは選んでDL」＝「I（私）が選ぶ」って語呂で覚えとこ。POPは全部ダウンロード（昔ながらの方式）、IMAPはサーバに残してタイトル確認しながら必要な分だけ取る（スマホ向き）。
💡 かんたん解説

POPは「全部の荷物をいっぺんに引き取る」方式。IMAPは「棚を見て必要な荷物だけ取る」方式。スマホとPCで同じメールを見たいならIMAP（サーバに残る）が便利。
