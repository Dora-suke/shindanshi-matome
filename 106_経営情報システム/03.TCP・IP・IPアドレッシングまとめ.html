🏠 IPアドレスはインターネット上の住所

📍 IPアドレスとは

インターネットに繋がる機器（PC・スマホ・サーバー）すべてに一意に割り当てられる番号。例：

192.168.1.10 ← 4つの数字を「.」で区切る（IPv4）

各数字は0〜255の範囲（＝8ビット）。4つで合計32ビット。

🏢 自宅とオフィスで使う「プライベートIP」

家やオフィスの中だけで使う内線番号みたいなIP。外に出るときはルータが翻訳してインターネット側のIP（グローバルIP）に変わる。

例：192.168.0.5 （家庭で一番よく見る形）

📨 プロトコル＝「通信の約束事」

TCP（Transmission Control Protocol）ざっくり：確実に届ける通信方式

「ちゃんと届いたか確認しながら送る」丁寧な配達員。
Webページ・メール・ファイル転送で使う。

UDP（User Datagram Protocol）ざっくり：速いが確認しない通信方式

「投げっぱなし、速さ優先」の配達員。
動画配信・音声通話・ゲームで使う。

💡
違いのコツ
TCP＝書留郵便（確実・遅い）／UDP＝普通郵便の投函（速い・確認なし）。生配信で少しくらい音が途切れても続けたいからUDP、メールは1文字も抜けないように届けたいからTCP。

🏢 通信の階層モデル（まずイメージ）

💡
イメージ先行
通信は「会社の部署分担」と同じ考え：
  • 営業部（アプリ）：顧客とやり取りする内容を決める
  • 企画部（転送）：どう届けるか方針を決める（確実 or 速さ）
  • 配送部（経路）：どの道を通るか決める
  • 運搬部（物理）：実際にケーブルや電波で運ぶ
この役割分担を標準化したのがOSI参照モデル7層と、実用版のTCP/IPモデル4層。

🏠 IPアドレスは「住所＋部屋番号」

IPアドレスの構造（32ビット）
ネットワーク部どの街か
（番地）
ホスト部その街の
どの家か

例：192.168.1．10 → 192.168.1の街の10番地の家

サブネットマスクざっくり：IPアドレスの「どこまでがネットワーク部か」を表す目印とは

「ネットワーク部がどこまでか」を教える相棒。例：255.255.255.0 なら先頭24ビットがネットワーク部、残り8ビットがホスト部。

CIDR表記ざっくり：サブネットマスクのビット数を「/24」のように短く書く方式（シーアイディーアール）：192.168.1.0/24 ＝「先頭24ビットがネットワーク部」の意味。

💡
まずイメージ

🔄 TCP vs UDP（使い分けが核心）

TCP

コネクション型
3ウェイハンドシェイクざっくり：通信開始前にSYN→SYN+ACK→ACKの3回やり取りして回線を確立で接続確立
順序制御・再送制御あり
確実だが遅い

用途：Web、メール、ファイル転送、SSH

VS

UDP

コネクションレス型
接続確立なし・投げっぱなし
再送なし
速いが信頼性なし

用途：動画配信、音声通話（VoIP）、オンラインゲーム、DNS

📝 【1次】試験ポイント：R02問13・R05問11など頻出。「TCPは順序制御・再送制御を行う」「UDPはコネクションレス」が核心。DNSがUDPを使う（速さ優先・再送不要）のも定番知識。

⚠️ 【1次】引っかけ：「UDPは再送制御で信頼性を確保する」は誤り。再送制御はTCP。UDPは再送しないからこそ速い。

🔄 NAT と NAPT（ネットの翻訳機）

💡
まずイメージ
家の中の内線番号（プライベートIP）は外では通じない。外に出るときルータが外線番号（グローバルIP）に翻訳してくれる。これが NAT／NAPT。

NAT（Network Address Translation）ざっくり：プライベートIPとグローバルIPを1対1で変換

プライベートIPを1対1でグローバルIPに変換。家のPCが複数あっても、同時に外に出られるのは1台だけ（グローバルIPが1個なら）。

NAPT（Network Address Port Translation）＝ IPマスカレードざっくり：ポート番号も使って多数のプライベートIPを1つのグローバルIPに集約

IP＋ポート番号も変換することで、1つのグローバルIPで複数のPC／スマホを同時にインターネット接続可能。家庭のWi-Fiルータはほぼこれ。

📝 【1次】試験ポイント：H21問15・R04問11など頻出。「1つのグローバルIPで複数端末を同時接続可能にするのは？」→NAPT（IPマスカレード）。NATは1対1、NAPTは多対1。

🚪 ポート番号（ウェルノウンポート）

ポート番号とは

「届いた荷物を家のどのアプリに渡すか」を示す番号。0〜65535の範囲で、0〜1023がウェルノウンポート（well-known port）として主要プロトコル用に予約されている。

プロトコルポート用途
HTTP80Web（非暗号化）
HTTPS443Web（暗号化）
FTP20／21ファイル転送
SSH22暗号化リモート接続
SMTP25メール送信
POP3110メール受信（POP）
IMAP143メール受信（IMAP）
DNS53名前解決
DHCP67／68IP自動配布

📝 【1次】試験ポイント：HTTP=80、HTTPS=443、SSH=22、SMTP=25 は頻出。特にHTTP/HTTPSの組合せは暗記必須。

🌐 DNS と DHCP

DNS（Domain Name System）ざっくり：ドメイン名（文字）をIPアドレス（数字）に変換するシステム

「google.com」→「142.250.196.142」のように名前をIPに変換。人間は名前、機械は数字、の橋渡し。

DHCP（Dynamic Host Configuration Protocol）ざっくり：IPアドレスなどネット設定を自動で配布する仕組み

PCがネットに繋がるとき、IPアドレス・サブネットマスク・デフォルトゲートウェイ・DNSサーバーを自動で設定してくれる。

📝 【1次】試験ポイント：「Wi-Fiに繋いだだけでネットが使えるのは？」→DHCPが設定を配布、DNSが名前解決してくれるから。セット暗記。

🔄 ルーティングとデフォルトゲートウェイ

🚗 郵便局の集配所＝ルータ

ルータは複数のネットワーク間の境界にいて、届いた荷物（パケット）を次にどこに送るか判断する。その判断テーブルがルーティングテーブル。

デフォルトゲートウェイざっくり：外のネットワークに出るときの最初の転送先（通常は家/社のルータ）＝「宛先が自分のネットワーク内になければ、とりあえずここに渡せ」というデフォルト転送先。

家庭なら 192.168.1.1（Wi-Fiルータ）がデフォルトゲートウェイ。

家のPC2台が同時にYouTubeを見る場合
端末送信前（プライベートIP：ポート）ルータで変換後（グローバルIP：ポート）
PC-A192.168.1.10 : 50001203.0.113.5 : 60001
PC-B192.168.1.11 : 50001203.0.113.5 : 60002

ルータがポート番号を付け替えて区別。返答が戻ってきたらポート番号で「どのPC宛か」判別して内部に戻す。

🐜 ARP と ICMP（縁の下）

ARP（Address Resolution Protocol）ざっくり：同一ネット内でIPからMACアドレスを問合せるプロトコル

IPアドレスだけ知っていても、同じLAN内で届けるにはMACアドレスざっくり：NIC（ネットワーク機器）に固有の物理アドレスが必要。ARPがLAN内に「このIPの人〜？」と呼びかけて対応するMACを返してもらう。

ICMP（Internet Control Message Protocol）ざっくり：通信状態の制御メッセージを送るプロトコル

通信エラーや到達確認のメッセージを運ぶ。pingコマンド（「届く？」→「届いた」の応答）はICMPを使う。

🧮 サブネット計算の実演

例題：192.168.1.100/26 のネットワークは？

Step 1：/26＝ネットワーク部26ビット、ホスト部は32−26＝6ビット

Step 2：ホスト数＝2^6−2＝62台

Step 3：1ブロックのサイズ＝2^6＝64。ブロックは0, 64, 128, 192 の4つ

Step 4：100は64〜127の範囲 → このサブネットは 192.168.1.64/26

  • ネットワークアドレス：192.168.1.64（末尾0は予約）
  • 利用可能IP：192.168.1.65 〜 192.168.1.126（62台分）
  • ブロードキャスト：192.168.1.127（末尾all 1は予約）

📝 【1次】試験ポイント：R02問9・R04問11でサブネット計算が出題。「与えられたIPがどのサブネットか」「ホスト数は何台か」が問われる。2^(32−CIDR)−2の式を押さえる。

⚠️ 【1次】引っかけ：「IPv6では NATが必須」は誤り。正はNATは不要（アドレスが枯渇しないため全端末にグローバルIP割当可）。

OSPFOpen Shortest Path First中〜大規模組織内回線速度を考慮して最短経路計算。収束速い
BGPBorder Gateway Protocolインターネット全体（ISP間）ポリシーベース。世界規模の経路制御

📝 【1次】試験ポイント：「インターネット全体（AS間）の経路制御は？」→BGP。「小規模LAN向きでホップ数ベースは？」→RIP。範囲と選択基準を対応させて暗記。

🏢 中小企業の事例で学ぶ

事例1

小売店A社：新店舗のネットワーク設計を頼まれた

与件：端末30台（レジ10＋事務20）＋IoTセンサー10台、将来20台増える可能性。

設計：/27（30台）では将来パンクする → /26（62台）採用。プライベートIP 192.168.10.0/26でDHCP配布。外向き接続はNAPTで1つのグローバルIPを共有。

学び：サブネット設計は現状＋将来を含めた台数で決める。NAPTでコスト削減。

事例2

製造業B社：自社Webサーバーへ社外アクセス設定

与件：外部からWebを公開したい。内部LANは192.168.0.0/24。

解決：ルータでポート転送（ポートフォワーディング）を設定。グローバルIPの443番宛パケットを、内部Webサーバー（192.168.0.10）の443へ転送。ファイアウォールは80/443のみ開放。

事例3

サービス業C社：テレワーク時の社内システム接続

与件：従業員が自宅から社内基幹に安全にアクセスしたい。

解決：VPN（Virtual Private Network）で暗号化トンネル構築。自宅PCが社内LANに居るかのように振る舞う。社内のDNS・プライベートIPで社内リソースへアクセス可能。

💡 実在サービスのストーリー

🌍 IPv4アドレス枯渇（2011年）

2011年、IANA（インターネット番号割当機関）のIPv4アドレス在庫が枯渇。アジア太平洋地域（APNIC）も同年枯渇。各国はNAPTで延命しつつ、IPv6への移行を推進中。日本では2011年にフレッツ光ネクストでIPv6が標準化。

学び：IPv4の32ビット（約43億個）では世界の端末数に足りない。IPv6の128ビットで「地球上の砂粒1個に1アドレス配れる」ほどの規模に。

💻 家庭のWi-FiルータはNAPTの塊

家にあるバッファローやNECのWi-Fiルータ。ISPから割り当てられるグローバルIPは1つだけ（動的）。なのに家中のスマホ、PC、スマート家電、Switch、Apple TVが同時に使える。これは全てルータがNAPTでポート番号を付け替えて1つのグローバルIPに集約しているから。

🔐 Let's Encrypt と HTTPS普及

2015年まではHTTPSの証明書が年間数万円と高価で、中小サイトはHTTPのままだった。Let's Encryptが無料でSSL/TLS証明書を発行する仕組みを提供したことで、HTTPSが爆発的に普及。2024年現在、主要ブラウザはHTTPサイトに警告表示する時代に。

🏢 AWS VPC：クラウドでも同じ概念

Amazon Web Services の仮想ネット（VPC）は、CIDRでアドレス範囲を定義し、サブネットに分割、ルーティングテーブルとNATゲートウェイで外部通信を制御。オンプレのネットワーク設計と完全に同じ考え方が使える。
