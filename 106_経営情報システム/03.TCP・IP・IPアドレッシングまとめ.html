🌐 インターネットの概要（教科書頻出 H22-8／H21-6／H18-10）

インターネット＝ネットワーク間のネットワーク

インターネット（Internet）は、世界中の無数の LAN が相互に接続された巨大なネットワーク。直訳すると「ネットワークの間のネットワーク」。

実際には、インターネットは TCP/IP というプロトコルに基づいたネットワーク。プロトコル＝通信の約束事。

特定の管理者が作ったわけではなく、TCP/IPに基づいて世界中の人が接続することで大きくなった。

🌍 ICANN（IPアドレス・ドメイン名管理）

現在、インターネット上の IPアドレスやドメイン名の割り当ては、ICANN（Internet Corporation for Assigned Names and Numbers）という非営利団体で管理されている。

ただし、基本的には自由に参加・利用ができるネットワーク。

🔍 WWW（World Wide Web）

インターネット上で、ページを作成・参照するための仕組みが WWW（略して Web）。WWWの登場と Web ブラウザの普及で、一般ユーザーにも爆発的に広がった。

インターネットが普及したことで、電子メールやファイル転送なども自由に行えるように。

📚 TCP/IP プロトコル（教科書頻出 R5-12／R1-12／H25-10／H20-8／H18-10）

TCP/IP プロトコルは、正確にはIP と TCP の2つのプロトコルで構成される。IP と TCP は別々の階層で動作。

プロトコルは階層構造になっており、情報を送る側は上位層から下位層へ、受け取る側は下位層から上位層へ流れる。

🏢 OSI 基本参照モデル（7階層）

OSI（Open Systems Interconnection）基本参照モデル

ISO（国際標準化機構）が定めた7階層からなるプロトコルの標準モデル。TCP/IP のような現実のプロトコルではないが、プロトコルの標準モデルを定義したもの。

層名称定義する内容
第7層アプリケーション層提供する通信サービスの種類（HTTP/FTP等）
第6層プレゼンテーション層通信で利用するデータの形式（文字コード変換等）
第5層セッション層通信の開始から終了までの管理
第4層トランスポート層通信における信頼性確保の方法
第3層ネットワーク層異なるネットワーク同士の中継方法
第2層データリンク層限られた範囲内での機器間の接続方式
第1層物理層物理的な接続方法（ケーブルなど）

🔌 OSI と主な接続機器

各層によって、主な接続機器が異なる：

層主な接続機器役割
第4〜7層ゲートウェイ異なるネットワーク同士を中継する仕組みの総称
第3層ルータ・L3スイッチIPアドレスを見て経路選択
第2層ブリッジ・L2スイッチMACアドレスを見て適切な宛先に転送
第1層リピータ・ハブ・ケーブルデータを流すための物理的基盤

📍 IP（Internet Protocol）（教科書最頻出 R6-9／R5-11／R4-8／R1-8／R1-11／H29-11ほか）

IP＝インターネットの基本

IP（Internet Protocol）は、通信する際のデータの最小単位であるパケットを、相手に送信する役割。

パケットは、データ本体と 宛先などのヘッダから構成される。

🏠 IPアドレス

IPアドレス＝ネット上の住所

パケットの宛先は IPアドレスで記載。ネットワークに接続される機器に一意の番号として割り当てる。

現在主流の IPv4 は4つに区切られた32ビットの数値。例：192.168.36.123（10進表記）

🏪 サブネットマスク

サブネットマスクは、IPアドレスをネットワーク部とホスト部に分けるための数値。2進法で 1の部分がネットワーク部、0の部分がホスト部。

例：ネットワークアドレス 192.168.10.0／255.255.255.0（24ビット） → 先頭24ビットがネットワーク部、残り8ビットがホスト部。

CIDR表記では 192.168.10.0/24 と書く（/24 は「先頭24ビットがネットワーク部」の意）。

📖 ドメイン名と DNS

ドメイン名

IPアドレスは数字の羅列で覚えにくい。そこで人間に分かりやすいドメイン名を付ける。例：studying.jp（スタディング）。

DNS（Domain Name System）

ドメイン名から IPアドレスへの変換は DNS（Domain Name System）というシステムで行う。世界中の DNSサーバが協力して動作。

📞 グローバルIP と プライベートIP

グローバルIP

全世界からのアクセスで使用するため、重複しない一意の番号が必要。

プライベートIP（ローカルIP）

社内ネットワークなど外部との接続が必要ない場合に、内部だけに通用するIPを割り当てる。内部で一意であれば良い。

🔄 NAT と NAPT（IPマスカレード）

NAT（Network Address Translation）

プライベートIP を グローバルIP に変換する仕組み。社内のPC が外部のインターネットにアクセスするためにルータが提供する機能。
NATは1対1で変換。5台のPCが同時にアクセスするには5個のグローバルIPが必要。

NAPT（Network Address Port Translation）＝IPマスカレード

プロバイダから割り当てられるグローバルIPは1つだけのケースが多い。NAPT は1つのグローバルIP を複数の端末で共有する仕組み。
ルータが各端末に異なるポート番号を割り当てて識別。NAT の一種とされる。

📝 【1次】試験ポイント：R1-11、H29-11 出題。「NAT=1対1／NAPT=多対1（ポートで識別）」。

NAT＝1対1NAPT（IPマスカレード）＝多対1

🔁 DHCP（Dynamic Host Configuration Protocol）

ネットワークに接続するコンピュータに IPアドレスなどを自動的に割り当てるプロトコル。
DHCPサーバを用意しておけば、ネットワーク接続時に自動で IPアドレスが割り当てられ、ネットワーク管理が簡単になる。

📝 【1次】試験ポイント：R4-8、R1-11、H19-14 出題。「IPアドレス自動割当＝DHCP」。

⭐ IP最重要点
  • ❶ IPv4＝32ビット／IPv6＝128ビット
  • ❷ サブネットマスク＝ネットワーク部とホスト部の境界
  • ❸ DNS＝ドメイン名→IP変換／NAT＝プライベート⇔グローバル変換／NAPT＝多対1（ポート識別）／DHCP＝IP自動割当

🚪 TCP（Transmission Control Protocol）（教科書頻出 R1-11／H29-11）

TCP＝IPの上の階層

IP では IPアドレスを宛先としてパケットを特定のコンピュータに送信できた。

TCP は、さらにポート番号を使用することで、コンピュータ上のアプリケーション（プログラム）を特定する。

TCP は到着の確認やエラーの訂正を行い、通信を正確に行うように制御する。

🏢 ポート番号＝建物の部屋番号

IPアドレスが建物の住所とすれば、ポート番号は部屋番号のようなイメージ。

ポート番号は0〜65535の数値。よく使われる上位プロトコルにはあらかじめ決まったポート番号が付く。
例：Webページ表示のHTTPプロトコルのポート番号は80。

📱 UDP（教科書補足）

UDP（User Datagram Protocol）

OSI 基本参照モデルでは、TCP と同じトランスポート層に相当。

TCPと違いコネクションレス型のデータ転送プロトコルで、到着確認などは行わない。
TCPよりも信頼性は低いが、転送速度が高い。

TCP＝信頼性高（書留）UDP＝速い（投げっぱなし）
⭐ TCP・ポートの最重要点
  • ❶ TCP＝アプリ特定（ポート番号）＋到着確認・エラー訂正
  • ❷ HTTP=80（ポート番号の代表例として教科書記載）
  • ❸ UDP＝コネクションレス・信頼性低・速い
  • ❹ 各層の識別子：MAC（NIC）／IP（コンピュータ）／ポート（アプリ）

📨 アプリケーション層のプロトコル（教科書頻出 R4-7／H21-6／H20-6／H19-5）

📖 HTTP（Hyper Text Transfer Protocol）

Webページの元データであるハイパーテキストを送受信するためのプロトコル。
Webブラウザでアクセスする際、「http:」または「https:」から始まる URL（Uniform Resource Locator：ネットワーク上のリソースの場所を表す識別子）を指定。

📂 FTP（File Transfer Protocol）

ファイルを転送する際のプロトコル。大きなファイルのダウンロードや、Webページを Webサーバに転送する時などに使用。

📧 電子メール関連：SMTP／POP／IMAP

  • SMTP（Simple Mail Transfer Protocol）：電子メールの送信
  • POP（Post Office Protocol）：電子メールの受信（全DL方式）
  • IMAP（Internet Message Access Protocol）：電子メールの受信（選択DL方式）

🌐 WWW と HTML（教科書頻出 H21-4／H20-5／H20-6）

WWW（World Wide Web）

インターネット上で Webページを参照できる仕組み。Webブラウザに URL を入力すると Webページを読み込んで画面に表示。
Webページを公開するには Webサーバが必要。

HTML（HyperText Markup Language）

Webページを作るための言語。文字だけでなくリンク・見出し・画像などを含めた Webページを記述できる。リンクをクリックすることで様々なページに移動可能。

HTML で作成した Webページは FTP などで Webサーバに転送し、公開する。

MIME（補足・教科書 R6-9／R4-7 出題）

MIME（Multipurpose Internet Mail Extension）は、電子メールで画像・音声・動画などの添付ファイルも送受信できるようにする仕組み。データをテキストデータに変換することで送受信可能に。

🔒 HTTPS（4-4 セキュリティ章への接続）

「https:」から始まる URL は HTTP + SSL/TLS で暗号化された通信。詳しくは別ノート（セキュリティ）で扱う。Webページの盗聴防止＋なりすまし防止を実現。
