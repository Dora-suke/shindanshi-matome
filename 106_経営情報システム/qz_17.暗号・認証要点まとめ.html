🔬 ハッシュ関数の性質

ハッシュ関数（ざっくり：データを固定長の指紋に変換する関数）

  • 固定長出力：入力長に関わらず出力は一定（例：SHA（Secure Hash Algorithm）-256＝256ビット）
  • 一方向性：ハッシュ値から元データは復元不可能
  • 衝突耐性：別の入力から同じハッシュは作りにくい
  • 用途：デジタル署名／パスワード保存／改ざん検知
アルゴリズム出力長評価
SHA（Secure Hash Algorithm）-256256bit✅ 推奨（現役）
SHA（Secure Hash Algorithm）-1160bit⚠️ 非推奨（衝突発見済）
MD5（Message Digest 5）128bit❌ 非推奨（脆弱）

⚠️【1次】「ハッシュ値から元データを復元できる」は誤り。一方向性が定義。

💡 かんたん解説

ハッシュは「データの指紋」。どんなに長いデータでも決まった長さの指紋に変わり、元データが1文字でも違うと指紋はまるごと別物になる。だから改ざんが一発でバレる。指紋から元データには戻せない（＝一方向）。

🔐 SSL/TLSの基本

  • TLS（Transport Layer Security）はSSL（Secure Sockets Layer）の後継。SSL3.0までで終わり、現在はTLS1.2/1.3が主流
  • HTTPS（HTTP over TLS）＝HTTP（HyperText Transfer Protocol）＋TLS。ポート番号は443
  • ハンドシェイク：①証明書検証 → ②共通鍵交換 → ③共通鍵で通信
ヨンヨンサンで安全HTTPS = 443SSLは引退・TLSが現役後継関係
💡 かんたん解説

SSL/TLS は「最初だけ丈夫な金庫（公開鍵）でナイショの合言葉を渡し、あとはその合言葉（共通鍵）で速くやりとりする」仕組み。遅いけど安全な公開鍵は鍵渡しの1回だけ、本番の通信は速い共通鍵に切り替える、といいとこ取り。

💡 かんたん解説

本人確認の方法は「知ってること・持ってるモノ・体の特徴」の3タイプ。MFA（多要素認証）は、このうちタイプの違うものを2つ以上組み合わせること。同じタイプを2つ（パスワード＋秘密の質問）並べても、種類が同じなのでMFAにならない。

🏠 家に入るとき：暗証番号（知識）だけだと忘れたり人に教えちゃうかも。鍵（所持）＋暗証番号（知識）の2種類が必要＝それがMFA（Multi-Factor Authentication・多要素認証）。

📌 MFAは「種類の違う鍵を2つ以上」。知識だけ2つ（パスワード＋秘密の質問）は種類が同じ→多段階でMFAではない。

⚠️ MFA（Multi-Factor Authentication・多要素認証）の定義

異なる要素を2つ以上組み合わせる。同じ要素を2つ重ねるのは「多段階認証」であって多要素ではない。

  • ✅ MFA（Multi-Factor Authentication・多要素認証）：パスワード（知識）＋スマホ（所持）
  • ❌ MFA（Multi-Factor Authentication・多要素認証）ではない：パスワード＋秘密の質問（両方とも知識）
  • ❌ MFA（Multi-Factor Authentication・多要素認証）ではない：パスワード＋PIN（Personal Identification Number）（両方とも知識）

⚠️【1次】R04新出。「パスワード＋秘密の質問でMFA」は誤り。要素が同じ（知識）なので多段階に過ぎない。

SSO（Single Sign-On・シングルサインオン）

1度のログインで複数システムにアクセス可能。SAML（Security Assertion Markup Language・セキュリティアサーションマークアップ言語）／OAuth（オープン認可）／OpenID Connectなどの規格で実現。

⭐ このタブの要点
  • ❶ 認証3要素＝知識（SYK：Something You Know）・所持（SYH：Something You Have）・生体（SYA：Something You Are）。MFA（Multi-Factor Authentication・多要素認証）は異なる要素を2つ以上
  • ❷ CHAP（Challenge Handshake Authentication Protocol・チャレンジハンドシェイク認証プロトコル）は暗号（チャレンジレスポンス）／PAP（Password Authentication Protocol・パスワード認証プロトコル）は平文（非推奨）。OTP（One-Time Password・ワンタイムパスワード）は1回限り（TOTP（Time-based One-Time Password）＝時間ベース）
  • ❸ SSO（Single Sign-On・シングルサインオン）破られると全システム陥落リスク。JPKI（Japan Public Key Infrastructure・公的個人認証サービス）は署名用（実印）＋利用者証明用（認印）の2種

① ハッシュアルゴリズム比較
アルゴリズム出力長現在の評価
SHA（Secure Hash Algorithm）-256256bit✅ 推奨（現役標準）
SHA（Secure Hash Algorithm）-1160bit⚠️ 非推奨（衝突発見済）
MD5（Message Digest 5）128bit❌ 非推奨（脆弱・衝突容易）

⚠️ 「ハッシュ値から元データを復元できる」は誤り（一方向性が定義）

② デジタル署名フロー（使う鍵の種類）
フェーズ操作使う鍵
送信側①平文をハッシュ化して指紋を作る（鍵は不要）
送信側②指紋を暗号化して「署名」を作る送信者の秘密鍵
受信側①署名を復号して指紋を取り出す送信者の公開鍵
受信側②受信平文のハッシュ値と照合（一致なら本物）

📝 署名の3効能：本人性・改ざん検知・否認防止　❌ 機密性は含まない

③ PKI（Public Key Infrastructure・公開鍵基盤）の構成要素
要素役割・内容
CA（Certificate Authority・認証局）公開鍵が本人のものと保証する信頼できる第三者機関
電子証明書「この公開鍵はXさん本人」とCAが署名した証明書
CRL（Certificate Revocation List・証明書失効リスト）失効した証明書の一覧リスト（定期更新・オフライン参照可）
OCSP（Online Certificate Status Protocol）証明書の有効性を1件ずつリアルタイムで照会する方式
④ HTTP vs HTTPS 比較
項目HTTP（HyperText Transfer Protocol）HTTPS（HTTP over TLS）
暗号化❌ なし（平文・筒抜け）✅ TLS（Transport Layer Security）で暗号化
ポート番号80443
盗聴・改ざん防げない（MITM攻撃が容易）防げる

📝 TLSハンドシェイクの順序：①証明書検証 → ②公開鍵で共通鍵を安全に交換 → ③共通鍵で本通信（高速）

⑤ 認証の3要素とMFA（Multi-Factor Authentication・多要素認証）
要素英略語具体例
知識（Something You Know）SYKパスワード・PIN・秘密の質問
所持（Something You Have）SYHICカード・スマホ・トークン
生体（Something You Are）SYA指紋・顔・虹彩・静脈
組み合わせMFA成立？
パスワード（知識）＋スマホ（所持）✅ 成立（異なる要素）
パスワード＋秘密の質問（両方とも知識）❌ 成立しない（同じ要素）＝多段階認証
指紋＋虹彩（両方とも生体）❌ 成立しない（同じ要素）
⑥ 個別認証手法の比較
方式仕組みのポイント安全性
CHAP（Challenge Handshake Authentication Protocol）サーバが乱数（チャレンジ）を発行し、クライアントがハッシュ計算して返す✅ パスワードが流れない
PAP（Password Authentication Protocol）平文でパスワードをそのまま送信する❌ 盗聴に弱い・非推奨
OTP（TOTP：Time-based One-Time Password）時間ベースで30秒ごとに自動更新される1回限りのパスワード✅ 盗聴されても再利用不能
