# 確認システム

ターミナル対話で「選択肢を絞れるレベル（記述ではなく識別）」を確認・サポートするための専用フォルダ。
ノートは参照のみ。ここには確認の進捗・識別ポイント・ログを蓄積する。

## 運用プロトコル
1. 優先順位順に論点を1つ出題
2. その論点の識別ポイント（過去問の選択肢由来の○×・ひっかけ）を5〜8問提示
3. ユーザーは正/誤（自信なければ△）で回答。説明不要、絞れるかだけを見る
4. 即採点＋外した区別・急所だけ一言解説
5. 論点を「クリア／要再訪」で締め、次へ

## 確認クイズUIの仕様（`確認_<番号>_<科目名>.html` 共通）
- **回答**：正/誤/△ を押すと即・正解と急所を表示。選んだボタンは黄枠で強調。
- **← 前へ**：テスト中いつでも前の問題に戻れる（Q1では無効）。戻った問題は正解・急所を表示したまま、○△× を**選び直して上書き**できる。記録される前なら何度でも修正可。
- **次へ →**：回答後のみ表示。最終問では「結果を見る →」になり完了画面へ。
- **🔄 最初からやり直す**：完了画面のボタン。Q1に戻し回答をリセット。再完了時はサーバが最新結果で上書き記録する。
- 論点を差し替えるときに編集するのは **SUBJECT / LOGIC / TOPIC / Q配列** のみ。上記ナビ（前へ／最初から）の仕組みは触らず温存する。

## 判定基準
- 到達目標 = 選択肢が出たとき正誤を見分けて絞れるレベル（≠白紙から記述）
- ◎=自信を持って絞れる ／ △=あいまい ／ ×=見分けられない
- 識別ポイントの大半が◎ → その論点クリア

## ファイル構成（科目ごとにフォルダ分割）
ルート直下は共通ファイルのみ。科目ごとの中身は `<科目番号>_<科目名>/` フォルダにまとめる。

```
確認システム/
├ index.html        … 開始メニュー（各科目・苦手復習へのリンク）
├ server.py         … ローカルサーバ（:8765 配信＋/saveで自動記録）
├ 開始.command      … サーバ起動＋メニューを開く
├ README.md         … 本ファイル（運用プロトコル）
├ 進捗.md           … 全科目横断の到達マップ（クリア/要再訪/未着手）
└ 103_企業経営理論/                       ← 科目フォルダ（新科目はこの形で追加）
   ├ 確認_103_企業経営理論.html           … 確認クイズ（論点を差し替え使い回し）
   ├ 記録_103_企業経営理論.md             … 各問の内容・正解・結果・傾向を蓄積（Claudeが読む）
   ├ 自動記録_103.md                      … サーバが解答結果から自動生成（最新で上書き）
   ├ _results_103.jsonl                   … 生ログ（append-only・機械可読）
   ├ 苦手復習_103_01_… / _02_… / _03_….html … 読んで覚える図解ノート
   └ images/                              … 苦手復習で使う画像
```

- 新しい科目を始めるときは `<番号>_<科目名>/`（例 `104_運営管理/`）を作り、上と同じ構成でファイルを置く。
- `server.py` は `/save` 受信時、科目番号に一致する `<番号>_*` フォルダへ `_results` と `自動記録` を書き込む（フォルダが無ければルート直下にフォールバック）。
- ファイル名・フォルダ構成を変えたら `index.html` のリンクも更新する。

## 結果コードの形式
`LOGIC [マーク列] ○n △n ×n / 要再訪:…`（マーク列は設問順 ○/△/×）
ユーザーはこの1行を貼るだけ → Claudeが記録mdと進捗mdを更新。
